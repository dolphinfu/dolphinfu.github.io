快乐的小谢

发表于 2020-06-30 分类于 github 阅读次数：
本文字数： 1k 阅读时长 ≈ 1 分钟

  我的朋友小谢是github的深度粉丝，他推荐我来github建个仓库建个站什么的，小谢是一个技术宅男兼钢铁直男，公司不鼓励加班，但他还是每天加班到半夜，在做嵌入式开发的同时自我驱动，兼职开发实用工具帮助其它工程师更好的工作。小谢的快乐很纯粹，每个人都能感受到的那种，写一段代码就可以乐半天，完善了一个工具就可以开心好几天。

  这次我准备搞一个Github的Page，我一个硬件工程师啥也不懂，小谢帮我搞了一个通俗易懂的攻略，我就跟着学了一下，还不错，挺有意思的，但是小谢同志给我弄了一个巨丑的画面，我看到的时候都惊呆了，内心有一种赶快放弃的想法，哎，小谢哪儿都好，就是不注重外在美，他做的界面也是每次都被领导吐槽，如图所示：

​

  程序图标永远是：

​

  似乎还活在上个世纪，内在美虽好但完全不注重外在的东西恐怕也是不可取的吧，我作为硬件工程师一直都知道虽然没什么毛用但画pcb还是要画的美观一些，也需要经常设计外壳、贴膜等等，好歹简洁、合理、清爽的审美观还是要有的吧。

  诗云：“神存富贵，始轻黄金。”可神未到富贵，不该轻视黄金，中国的传统美德是注重内在修养，提升个人品德，当然是正确的，不要人夸颜色好，要留清气满乾坤，就像小谢一样努力建设仓库，码代码提升自身技术水平，这是一件很快乐的事，但在做这些的同时，完善一下个人审美、个人形象并不矛盾，须知再好的后端也需要前端的展示，再精湛的数据库也离不开出色的用户界面，不然技术再扎实甲方人员可能只看了一秒就请你黯然走人了，我们都是社会中的个体，现代社会只怕已经很难脱离社会去学陶渊明那样修养身心，我们终究需要包装一下自己。

  同时粗糙的前端能力对自身职业发展也是极其不利的，今天都已经是2020年了，技术没有太大瓶颈，说白了你会的总有其他人会，攻略、视频已经多的目不暇接，名牌大学、专业培训每年出品的编程技术人员数以万计，csdn上技术博客到处都是，这年头最不缺的就是码农了吧，基础岗位或许还很多，但领导岗位越来越稀缺，和你竞争的往往也都是多年996的精英，如何脱颖而出？如何能更进一步？在修炼内功的同时，也许你还需要一些外功，更何况越上层的职位显然更需要整体思考的能力。

  最后，当然希望小谢快乐常在，早日找到自己的真爱。
